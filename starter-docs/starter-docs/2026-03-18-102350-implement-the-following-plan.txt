⏺ ★ Insight ─────────────────────────────────────
  Key design decisions in this skill:
  1. Team sizing rules scale dynamically (2-4 assessors) based on input count,
  avoiding the document-integrator's fixed 2-agent model. This prevents context
  overflow for large sets and under-utilization for small ones.
  2. The 9 Synthesis Rules are embedded directly in the Synthesizer agent's
  prompt rather than as separate reference material. This is intentional — the
  synthesizer needs these rules in its working context, not as a file to look
  up.
  3. Technique Fusion (Rule 3) is the most subtle rule: "absorb effects without
  labeling techniques." This prevents the common failure mode where a merged
  prompt reads like "Now using CoT: think step by step. Now using Role-Persona:
  you are an expert..." — instead, the techniques become invisible
  infrastructure.
